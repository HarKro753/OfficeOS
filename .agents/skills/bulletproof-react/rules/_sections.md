# Sections

This file defines all sections, their ordering, impact levels, and descriptions.
The section ID (in parentheses) is the filename prefix used to group rules.

---

## 1. Project Structure (structure)

**Impact:** CRITICAL
**Description:** Project structure determines developer velocity, onboarding speed, and how well the codebase scales with team size. A feature-based architecture with clear boundaries prevents the "big ball of mud" anti-pattern.

## 2. Project Standards (standards)

**Impact:** HIGH
**Description:** Automated standards enforcement (linting, formatting, type checking) eliminates entire categories of bugs and code review friction. Configure once, enforce everywhere.
