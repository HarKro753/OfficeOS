# Double Diamond Ideation Canvas

Use this canvas when the user wants a full guided ideation session or a reusable worksheet.

The process has four movements:

1. **Discover:** Go broad on audiences, moments, pains, alternatives, and market changes.
2. **Define:** Choose the strongest problem wedge.
3. **Develop:** Go broad again on solution shapes, business models, trust mechanisms, and distribution.
4. **Deliver:** Specify one testable concept with scope, exclusions, and validation.

## 1. Discover: Broad Map

Fill these sections before choosing.

### Raw Idea

```markdown
**Rough idea:**
**Why it feels interesting:**
**What is unclear:**
**Known constraints:**
```

### Audience Map

List 8-15 possible audiences.

For each:

```markdown
**Audience:**
**Why they might care:**
**How often the problem appears:**
**Current workaround:**
**Buying power or adoption path:**
```

### Painful Moment Map

List moments, not generic problems.

Use prompts:

1. When does the user feel stuck?
2. When does delay become expensive?
3. When does quality matter?
4. When does trust break?
5. When does the user have to coordinate too many people or tools?
6. When does the current process become embarrassing or reputation-risky?
7. When does the user repeat the same painful workflow?

### Alternatives Map

For each audience or moment, name what they do today:

1. Existing software.
2. Agency.
3. Freelancer.
4. Internal employee.
5. Spreadsheet/document/manual process.
6. AI tool.
7. Doing nothing.
8. Asking a friend/expert.

### Broken Tradeoff Map

Describe the cost of the current alternative:

1. Too slow.
2. Too expensive.
3. Too hard to trust.
4. Too much coordination.
5. Too technical.
6. Too generic.
7. Too unreliable.
8. Too manual.
9. Too risky.
10. Too low status or unprofessional.

## 2. Define: Problem Wedge

Choose 3-5 candidate wedges and fill:

```markdown
## Candidate Wedge: [Name]

**Customer:**
**Painful moment:**
**Current alternative:**
**Broken tradeoff:**
**Desired outcome:**
**Why this might be urgent:**
**Why this might be reachable:**
**What makes it weak:**
```

Then score each candidate with `references/scoring-rubric.md`.

## 3. Develop: Solution Possibilities

For the top 1-2 wedges, generate multiple solution shapes.

Use at least 5:

1. Productized service.
2. AI workflow tool.
3. Vertical SaaS.
4. Managed marketplace.
5. Verification/QA layer.
6. API/infrastructure.
7. Monitoring/alerting.
8. Expert network.
9. Template/playbook system.
10. Concierge MVP.

For each solution shape:

```markdown
**Shape:**
**Customer input:**
**Output/result:**
**What it removes from the customer's life:**
**Trust mechanism:**
**Smallest credible version:**
**Main risk:**
```

## 4. Deliver: Specific Concept

Finish with one concept.

```markdown
# Final Concept

**One-liner:**

**Customer:**

**Painful moment:**

**Current alternative:**

**Broken tradeoff:**

**Product promise:**

**First wedge:**

**Not for now:**

**Trust mechanism:**

**Why now:**

**Fastest validation test:**

**What would prove this wrong:**
```
